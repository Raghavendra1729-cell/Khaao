Canteen Pre-Order App — Technical Brief

MVP v1 · features · data model · APIs · stack · Docker + student-pack deployment

1. Core Model

The shopkeeper cooks to total demand, not per student. Finished units land in a Done Pool and the system matches them back to orders (FIFO). When an order is fully matched → student is notified to pick up and pay at the counter. No in-app payment, no OTP, no scheduling in v1 (order-now only). One active order per student; add items to it instead of placing new orders. No system cap on the shopkeeper — he throttles via accept / reject / trim.

4. The Pool Engine (core logic)

This is the only non-CRUD part — worth designing on paper first. Order & item states:

EntityStates
Ordersubmitted → accepted → preparing → (partially_ready) → ready → picked  | branches: rejected, expired
Order itemqueued → done → allocated → handed_over
  • Aggregate: Prep List = sum of queued item quantities across accepted orders.
  • Mark done: each “Done” adds one unit of that item to the Done Pool.
  • Allocate: assign pool units to orders first-order-first (by created_at).
  • Complete: when all of an order’s items are allocated → set ready, notify, set expires_at = now + 15m.
  • Add re-opens: a new item drops the order back to preparing until everything is ready again.
  • Expiry job: a ticker returns units of expired orders to the pool and sets expired.
  • Lock: once ready/picked, no new items may be added.

9. Out of Scope for v1

In-app payments · OTP · guest scan-to-order · time-slot scheduling · strike/ban system · bilingual toggle · analytics.

10. Build Order

  1. Foundation — Postgres schema + Go API skeleton + two-role auth.
  2. Menu — shopkeeper CRUD + availability; student browse.
  3. Ordering — place order; accept / reject / trim.
  4. Pool engine — Prep List, mark-done, FIFO allocation, completion.
  5. Real-time — SSE status on both sides + sound alerts.
  6. Hold + expiry — 15-min timer + return-to-pool ticker.
  7. Add-to-order — append + re-open logic.
  8. Ship — Dockerize, deploy on the VM, .tech domain + HTTPS, pilot one canteen.
